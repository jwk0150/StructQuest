# 树和二叉树

## 学习目标

掌握树和二叉树的定义、性质、存储结构和遍历算法；理解线索二叉树、哈夫曼树、AVL 树、红黑树、B 树、B+ 树和并查集的核心思想。

## 树与二叉树

树是 n 个结点的有限集，有唯一根结点，其余结点分为若干互不相交的子树。二叉树中每个结点最多有两个孩子，且左右子树有序。

二叉树遍历包括前序、中序、后序和层序。前序是根左右，中序是左根右，后序是左右根，层序通常借助队列。遍历序列可用于表达式树、语法树和目录结构。

## 哈夫曼树

哈夫曼树是带权路径长度最小的二叉树。构造时每次选权值最小的两个结点合并，体现贪心思想。哈夫曼编码是前缀编码，适合数据压缩。

## 平衡树

AVL 树要求任意结点左右子树高度差不超过 1，通过 LL、RR、LR、RL 旋转维持平衡。红黑树用颜色规则保证近似平衡，工程中插入删除代价更稳定。

B 树和 B+ 树是多路平衡查找树，适合外存和数据库索引。B+ 树数据集中在叶子结点，叶子间有链表，范围查询效率高。

## 并查集

并查集维护不相交集合，支持查找和合并。路径压缩和按秩合并后，操作复杂度接近常数，常用于连通分量和 Kruskal 算法。

## 易错点

二叉树不是度为 2 的树；二叉树可以为空，且左右孩子位置有区别。AVL 旋转要先判断失衡类型，再选择单旋或双旋。
