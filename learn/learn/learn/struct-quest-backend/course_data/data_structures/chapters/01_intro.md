# 绪论与算法分析

## 学习目标

理解数据结构的研究对象，区分逻辑结构、存储结构和数据运算；掌握抽象数据类型 ADT 的表达方法；能够使用时间复杂度和空间复杂度评价算法效率。

## 核心概念

数据结构是计算机组织和处理数据的方法。逻辑结构描述数据元素之间的关系，常见类型包括集合、线性结构、树形结构和图结构。存储结构描述逻辑关系在内存中的实现方式，常见类型包括顺序存储、链式存储、索引存储和散列存储。

抽象数据类型 ADT 将“数据对象、数据关系、基本操作”封装为一个逻辑模型。ADT 强调接口和语义，不依赖具体语言实现。例如线性表 ADT 可以定义 `insert(i, x)`、`delete(i)`、`get(i)` 等操作，而顺序表和链表只是不同实现。

算法是解决问题的有限步骤序列，通常具有有穷性、确定性、可行性、输入和输出。算法分析关注输入规模扩大时资源消耗的增长趋势。

## 复杂度

时间复杂度用大 O 记号表示增长上界。常见量级包括 `O(1)`、`O(log n)`、`O(n)`、`O(n log n)`、`O(n^2)`。空间复杂度表示算法运行过程中额外占用的存储空间。

## 例题

顺序查找长度为 n 的数组，最坏情况下需要比较 n 次，时间复杂度为 `O(n)`；二分查找每次缩小一半范围，时间复杂度为 `O(log n)`，但要求数据有序且支持随机访问。

## 易错点

不要把逻辑结构和存储结构混为一谈。线性表是逻辑结构，顺序表和链表是实现方式。不要把大 O 当成精确运行时间，它描述的是规模增长趋势。
