# 查找

## 学习目标

掌握顺序查找、折半查找、分块查找、二叉排序树、平衡树、哈希查找和 B+ 树索引的基本原理，能够分析平均查找长度和复杂度。

## 线性与有序查找

顺序查找逐个比较，适合顺序表和链表，不要求数据有序，复杂度为 `O(n)`。折半查找要求有序且支持随机访问，每次排除一半元素，复杂度为 `O(log n)`。

分块查找将数据分为若干块，块间有序、块内无序。先查索引表确定块，再在块内顺序查找，是顺序查找和折半查找之间的折中。

## 树表查找

二叉排序树满足左子树小于根、右子树大于根。理想情况下查找复杂度为 `O(log n)`，但若插入序列有序，可能退化为链表。AVL 树和红黑树用于维持平衡。

B 树和 B+ 树减少磁盘 IO，是数据库和文件系统常见索引结构。B+ 树叶子结点链表使范围查询特别高效。

## 哈希查找

哈希表通过哈希函数将关键字映射到表地址，平均查找可接近 `O(1)`。冲突处理包括开放地址法和链地址法。装填因子越高，冲突概率通常越大。

## 易错点

折半查找不能用于普通链表，因为链表不支持高效随机访问。哈希表平均很快，但最坏情况下仍可能退化。
