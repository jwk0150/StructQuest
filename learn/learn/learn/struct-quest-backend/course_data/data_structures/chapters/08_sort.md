# 排序

## 学习目标

掌握插入、交换、选择、归并、基数等排序算法；能够比较时间复杂度、空间复杂度、稳定性和适用场景。

## 插入排序

直接插入排序将待排序元素插入到已排序区间，适合小规模或基本有序数据。希尔排序按增量分组插入，是直接插入排序的改进，但通常不稳定。

## 交换排序

冒泡排序通过相邻元素交换逐步把最大或最小元素移到边界，稳定但效率较低。快速排序选取枢轴划分区间，平均复杂度 `O(n log n)`，最坏 `O(n^2)`，通常不稳定。

## 选择排序

简单选择排序每轮选择最小元素放到前面，交换次数少但不稳定。堆排序利用堆结构反复取最大或最小元素，复杂度 `O(n log n)`，空间 `O(1)`，不稳定。

## 归并与基数排序

归并排序采用分治思想，将两个有序序列合并，时间复杂度稳定为 `O(n log n)`，空间复杂度 `O(n)`，稳定。基数排序按关键字位分配和收集，适合整数、字符串等多关键字场景。

## 综合比较

稳定排序包括直接插入、冒泡、归并和基数排序。平均性能较好的通用排序包括快速排序、堆排序和归并排序。数据基本有序时，插入排序可能非常高效。

## 易错点

稳定性指相等关键字的相对顺序是否保持。不要只背复杂度，还要结合数据规模、初始有序程度、内存限制和是否需要稳定性选择算法。
