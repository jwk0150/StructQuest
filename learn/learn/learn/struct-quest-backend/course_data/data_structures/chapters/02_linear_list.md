# 线性表

## 学习目标

掌握线性表的逻辑特征，理解顺序表、单链表、双向链表、循环链表和静态链表的存储方式，能够分析插入、删除、查找等操作的复杂度。

## 核心概念

线性表是由 n 个数据元素组成的有限序列，除第一个元素外，每个元素有唯一前驱；除最后一个元素外，每个元素有唯一后继。线性表强调一对一的线性关系。

顺序表使用连续内存存储元素，支持按下标随机访问，读取第 i 个元素为 `O(1)`。缺点是中间插入和删除需要移动元素，平均复杂度为 `O(n)`。

链表用结点保存数据和指针。单链表结点通常包含 `data` 和 `next`。链表插入删除只需修改局部指针，但查找第 i 个结点必须从头遍历，复杂度为 `O(n)`。

双向链表增加 `prior` 指针，便于双向遍历和删除前驱后继。循环链表让尾结点指向头结点，适合循环访问场景，如约瑟夫问题。

## 代码示例

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.next = None

def insert_after(node, value):
    new_node = Node(value)
    new_node.next = node.next
    node.next = new_node
```

## 易错点

链表插入时应先让新结点指向后继，再修改前驱指针。删除结点时要保存被删结点的后继，避免链断裂。顺序表和链表没有绝对优劣，应按访问模式选择。
